ubuntu系统简单操作

  • 添加到收藏夹——类似windows固定到任务栏

  • win+空格键——输入法切换，shift键也可

  • CTRL+alt+t，终端打开

  • 简单命令

    • ls，简单查看当前所有目录内容
    • cd ..回到上一级目录
    • mkdir 创建新的目录
    • cd ~回到主文件夹
    • tab键，补全指令内容，且只有唯一内容
    • gedit 文本编辑器
    • source 执行文件中的指令序列 .sh文件
    • echo 显示的意思
    • .bashrc 环境变量的赋值
    • gedit ~/.bashrc 编辑主文件夹下的bashrc文件
    • ls-a 显示所有文件
    • sudo 以管理员权限执行这条指令

  • 在 ~/catkin_ws/src目录下执行git clone 后面添加GitHub项目网址

  • scripts 目录用于放置脚本文件和Python程序 安装依赖包

  • 安装工作空间内 的依赖包后，编译完成后，在工作空间文件夹下执行source指令

  • 使用source指令载入工作空间的环境设置

    • source ~/catkin_ws/devel/setup.bash

gedit ~/.bashrc

    • 加载的就是apt下载的那些软件包地址

  • source ~/catkin_ws/devel/setup.bash
    • 对应的是catkin工作空间里的软件包地址

  • 尽量选择系统中存在的软件包

  • CmakeList.txt文件中 Build部分

    • 声明一个C++可执行文件
    • 为这个包添加一个可执行程序
    • 可执行文件与包的名字保持一个名字

    • CTRL+shift +B进行编译

  • 运行节点

    • 运行ros roscore
    • 添加环境配置
    • rosrun

  • ctrl + shift +w

节点类型是在包中已经存在的，并且由该包的作者指定。

在ROS中，包是功能模块的基本单位，通常会包含一个或多个节点，还可能包含其它ROS特定的文件，如消息定义、服务定义、参数配置等。

每个节点都有一个特定的类型，类型指定了节点的实现方式，例如turtlesim_node就是turtlesim包中预定义的节点类型，用于仿真一只乌龟。节点类型可以在节点的C++源文件中指定，也可以在CMakeLists.txt文件中进行指定。

在ROS中，节点的输出可以分为两种类型：输出到终端和输出到ROS网络中的话题。output属性可以指定节点输出的类型，其可选的值包括：

log：输出到节点的日志中（默认值）。

screen：输出到终端上。

both：同时输出到节点日志和终端上。

在这里，output="screen"表示将节点的输出输出到终端上，方便调试。

​ args标签用于向节点传递命令行参数。在启动节点时，可以使用args标签传递任意数量和类型的参数。

这些参数在节点启动时会被传递给节点，并且可以在节点内部进行访问和使用。

在上面的例子中，turtle_tf_broadcaster节点使用args标签传递了一个参数，即/turtle1或/turtle2，用于指定广播的TF坐标系名称。turtle_tf_listener节点没有使用args标签，因为它不需要传递任何参数。
