ROS学习—Publisher发布者的python实现

  • ctrl+alt+t打开终端

  • 打开Vs code

    • 打开工作空间src目录，打开新创建的ssr_pkg

    • ssr_pkg软件包内src是存放c++节点代码，新建文件夹命名“scripts”，文件夹内新建文件“chao_node.py”,即python文件

    • ps

      • C++发布节点框架
        • 选择ROS管家NodeHandle
        • 向ROS管家NodeHandle申请初始化节点
        • 告诉管家NodeHandle需要发布的话题名称，索要能够发布消息包的发布对象pub
        • 开启while循环，不停使用Pub对象发布消息包

  • python框架

    • 选择ROS管家rospy

    • 向ROS管家rospy申请初始化节点

    • 告诉管家rospy需要发布的话题名称，索要能够发布消息包的发布对象pub

    • 开启while循环，不停使用Pub对象发布消息包

    • #!/usr/bin/env python3
#coding=utf-8
import rospy
from std_msgs.msg import String
if _name_== "_main_":
    rospy.init_node("节点名称")
    rospy.logwarn("消息内容") #表明初始化节点成功
    
    pub = rospy.Publisher("话题名称",String,queue_size=10)#String-话题消息类型，queue_size话题发布端的消息包缓冲长度
    
    #定义发布对象频率
    rate =rospy.rate(10)
    
   # 构建while循环，存在条件是节点没有被关闭
    while not rospy.is_shutdown():
        rospy.loginfo("  ")#预告下一步动作
        msg = String() #构建一个String消息包类型
        msg.data="  "#消息数据 
        pub.publish(msg)#发布到话题当中通过pub对象
        rate.sleep()#控制发布频率

    chmod +x file name

ROS机器人运动控制

  • 矢量运动控制和旋转运动控制

    • 对速度进行量化

    • 选择机器人底盘在地盘投影的中点

    • 建立三维坐标系，食指指向X轴，中指指向y轴，拇指指向z轴。三个方向的矢量合成

    • 沿x轴旋转正方向滚转运动，y轴俯仰运动，z轴偏航运动

    • ROS中速度控制即是矢量运动和旋转运动的三轴分量

  • 流程控制

    • 速度控制节点—速度控制话题—机器人核心节点
    • 速度控制节点消息类型：geometry_msgs/Twist
    • 速度控制话题：/cmd_vel

机器人运动控制的C++实现

  • 框架实现思路

    • 构建一个新的软件包，vel_pkg

    • 在软件包中新建一个节点，节点名字叫做vel_node

    • 在节点中，向ROS大管家NodeHandle申请发布话题/cmd_vel,并拿到发布对象vel_pub

    • 构建一个geometry_msgs/Twist类型的消息包vel_msg,用来承载要发送的速度值

    • 开启一个while循环，不停的使用vel_pub对象发送速度消息包vel_msg

    • cd catkin_ws/src/
catkin_create_pkg vel_pkg roscpp rospy geometry_msgs //geometry_msgs包含了速度消息类型的软件包
